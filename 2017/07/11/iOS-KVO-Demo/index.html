iOS KVO Demo

Veröffentlicht am 2017-07-11

1.KVO和通知（Notification）同属于观察者模式。
2.观察者模式的定义：
观察者模式定义了一种一对多的依赖关系，让多个观察者对象同时监听某一个主题对象。这个主题对象在状态上发生变化时，会通知所有观察者对象，使它们能够自动更新自己。

代码示例：

首先创建一个 ZRPerson 类

@interface ZRPerson : NSObject

@property (nonatomic, copy) NSString *name;

@property (nonatomic, assign) NSInteger age;


@end

在 ViewController 中初始化 和 属性赋值：

self.zrPerson = [[ZRPerson alloc] init];

self.zrPerson.name = @"zhangsan";

self.zrPerson.age = 18;

//设置监听。。。
// KeyPath 与 Key 虽然都是通过键去找值 但是 KeyPath 会自动寻址（一层一层的 valueForKey 下去）
[self.zrPerson addObserver:self forKeyPath:@"age" options:NSKeyValueObservingOptionOld | NSKeyValueObservingOptionNew context:nil];

//第一次改值
self.zrPerson.age = 81;//点语法（setter）相当于 setValue forKey

//第二次改值
dispatch_after(dispatch_time(DISPATCH_TIME_NOW, (int64_t)(3 * NSEC_PER_SEC)), dispatch_get_main_queue(), ^{

[self.zrPerson setValue:@88 forKey:@"age"];//
});

监听回调：

- (void)observeValueForKeyPath:(NSString *)keyPath ofObject:(id)object change:(NSDictionary<NSKeyValueChangeKey,id> *)change context:(void *)context {


if ([keyPath isEqualToString:@"age"]) {

NSLog(@"%@",[object valueForKey:@"age"]);

NSLog(@"change___:%@",change);
}

}

打印的change：

Change:{
kind = 1;
new = 81;
old = 18;
}

Change:{
kind = 1;
new = 88;
old = 81;
}

总结：
KVO 底层实现原理是系统给当前类创建子类 , 在子类 setter 方法调用父类的 setter 方法。通过修改 isa 指针指向系统创建的子类, 实现当前类属性值改变的监听。
